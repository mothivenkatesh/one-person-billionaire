# /plan-week — Design your operator's week

Runs the [`weekly-cadence-designer`](../skills/weekly-cadence-designer/SKILL.md) skill end-to-end to design (or audit) your weekly operating cadence.

## Use this when

- You feel reactive (calendar full of other people's needs)
- You're working 7 days and shipping nothing
- After hitting any major transition ($10K MRR, hiring, etc.)
- Quarterly cadence reset

## Workflow

This command runs one skill (no chain) but does it thoroughly. The skill enforces:

- 4-day work week (Mon-Thu) by default
- Public deadlines as forcing functions
- Default-no meeting policy
- Sunday redesign ritual

### Output

A complete cadence including:
- Calendar template (recurring blocks)
- Meeting policy doc (publishable)
- 4 forcing functions (Friday post / weekly customer call / monthly metrics / quarterly retro)
- Friday review process (3 hours)
- Sunday redesign ritual (60 min)
- Day-30 check-in plan

## After this command

- Week 1: cancel meetings flagged for removal; add new blocks
- Day 30: did deep work hit 50%? Did 4-day week hold? Did Friday post ship every week?
- If yes: continue. If no: re-run with adjustments.

## Pairs with

- `/annual-review` — for the bigger-picture 10-year statement
- `/diagnose-stall` — if your week is full but MRR is flat
