# Skill 14: Program Measurement Without a North Star Metric

**Category:** Operations | **Phase:** 5. Systematize

> **Key Principle:** Don't force one metric on diverse community programs. Each program gets its own goals. Roll them all up to two high-level objectives: top-of-funnel growth and user education.

## What It Is

A measurement framework for community programs where no single KPI captures the full picture. Instead of one North Star metric, each program is measured with its own specific goals that all feed into two high-level objectives.

## How Ben Did It at Notion

Notion's community operation had many programs running simultaneously: template gallery, ambassador program, decentralized community groups, influencer partnerships, certification program, champions community, events. Each of these programs was fundamentally different in nature, audience, and output.

Ben's team did NOT try to force them all under one metric. Instead, each program had its own measurement framework, and they all rolled up to two high-level goals.

## The Two High-Level Goals

Everything Notion did related to community served one or both of these:

1. **Top-of-funnel growth** — Drive more users to discover and sign up for Notion
2. **User education** — Help users learn to use Notion better and more deeply

Every program should clearly connect to at least one of these.

## Program-Specific Metrics

### Template Gallery
- Number of templates submitted and published
- Page views per template (SEO traffic)
- Signups attributed to template discovery
- Connected to: top-of-funnel growth

### Ambassador Program
- Percentage of ambassadors active each month
- Types of contributions (events organized, content created, templates submitted, community posts)
- Retention: how many renew for year 2?
- Connected to: both top-of-funnel growth AND user education

### Decentralized Community Groups
- Monthly member count per group
- Growth rate by region and language
- Which groups are thriving versus stagnating
- Connected to: both top-of-funnel growth AND user education

### Influencer Program
- Signups per creator partnership (tracked via UTM links)
- Cost per signup by creator tier and platform
- Creator retention (do they come back for another partnership?)
- Connected to: top-of-funnel growth

### Certification Program
- Number of certified consultants
- Enterprise referrals through the directory
- Consultant activity and success stories
- Connected to: user education

### Champions Community (Enterprise)
- Engagement rate in the community
- Feature adoption among champion accounts
- Satisfaction or NPS of champion users
- Connected to: user education

### Events
- Number of community-organized events per month
- Attendance per event
- Geographic distribution
- Connected to: both goals

## Step-by-Step SOP

### Step 1 — Map Each Program to High-Level Goals
For every community program, clearly articulate whether it drives top-of-funnel growth, user education, or both. If a program doesn't clearly connect to either, question whether it should exist.

### Step 2 — Define 2-3 Specific Metrics Per Program
Choose metrics that are trackable and actionable. Don't measure everything. Pick the 2-3 numbers that best indicate whether the program is healthy and growing.

### Step 3 — Set Up Tracking
Build dashboards or spreadsheets for each program. Make the data visible to the team. Ben's team built their own internal tracking tool for community groups.

### Step 4 — Review Monthly
Look at each program's metrics monthly. Ask: Is this growing? Is it healthy? Does it need more resources? Should we double down or reconsider?

### Step 5 — Let Programs Compete for Resources
When deciding where to invest more, compare programs by their impact on the two high-level goals. This creates a natural prioritization framework without needing one universal metric.

## Key Insight from Ben

The hardest thing was figuring out whether they were working on the right things. With limited resources, every program competed for attention. Having program-specific metrics that rolled up to two clear goals helped make prioritization decisions, but Ben acknowledged this was an ongoing challenge, not a solved problem.
